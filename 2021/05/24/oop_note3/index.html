函数对象

给函数赋值

void increase(int &x){x++;}
void (*func)(int&);//声明指针
func=increase;
int x=1;
func(x);
//可以使用auto推断函数名类型
auto func = increase;
//需要函数指针指向一个已有的函数

函数也可以作为变量

bool compare(int a,int b){
    return a>b;
}
std::sort(a,a+6,compare);//compare是一个函数指针
//以函数对象作为函数指针，要带上（），并且要求是public权限
//sort函数的第三个参数类型
template<class Iterator, class Compare>
void sort(Iterator first, Iterator last, Compare come){
    ...;//其中调用come时像函数一样使用即可
}

自定义类型的排序：

  1. 重载小于运算符
  2. 自定义比较函数
  3. 自定义比较函数对象

程序设计时，有可能整个算法的每个步骤都有多种选择

设计方法：

  1. 基于虚函数的模板设计模式，用含虚函数的类作为基类，再继承出派生类
  2. 使用函数指针作为参数来设计函数，或者使用函数对象作为模板内的类型，模板既可以接受函数指针，也可以接受函数对象。

函数指针和函数对象的统一：

#include <functional>
function<void(int,double)> add[]={func1,func2()};
//func1是指针,func2()是函数对象
//function是一个模板类

function也可以单个被赋值，仅仅根据函数返回值类型和参数类型来确定function的类型

智能指针

指针的销毁问题（多个指针指向一个对象，应该在所有指针都销毁后再销毁对象）

shared_ptr 来自于库

#include <memory>
//构造
shared_ptr<int>p1(new int(1));
shared_ptr<Class1> p2=make_shared<Class1>(233);
shared_ptr<Class1> p3=p2;
shared_ptr<int> p4;//空指针
//访问
int x=*p1;
int y=p2->data;//访问对象的某个变量

实现思路：

采用一个辅助指针指向真正的对象，然后对外部指针进行计数，计数为0则删除辅助指针和对象。

实际设计时，辅助指针设置为一个类，外部指针为成员变量为辅助指针的类，为了方便访问，外部指针类为辅助指针类的友元。

设计时，要考虑一个外部指针通过赋值改变类型的行为。

shared_ptr的其他用法

p.get();//获取裸指针
p.reset();//清除指针并且减少引用次数

不能使用同一个裸指针初始化多个智能指针，原因：裸指针的构造函数当中，会构造一个辅助制造。为避免此，构造第二个智能指针要求使用第一个指针来赋值。

不能直接使用智能指针维护对象数组，因为数组的销毁方式有[]。

弱引用：避免两个对象内的指针相互指向而无法销毁，造成内存泄露。

弱引用可以指向对象但不计数

shared_ptr<int> sp(new int(2));
weak_ptr<int> wp1=sp;
//若引用指针需要用智能指针来赋值
//其他用法
wp.use_count();//获取引用计数
wp.reset();//清除指针
wp.expired();//检查对象是否无效
sp=wp.lock()//从弱引用获得一个智能指针

独享所有权的指针：保证一个对象只能被一个指针引用，使用unique_ptr

#include <memory>
#incldue <utility>
using namespace std;
int main(){
    auto up1=std::make_unique<int>(12);
    unique_ptr<int> up2=up1;//错误，不能复制
    unique_ptr<int> up2=std::move(up1);//可以移动
    int *p=up2.release();//放弃控制权，返回一个指针
    delete p;
    return 0;
}
